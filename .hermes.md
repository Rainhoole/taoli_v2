# TAOLI — Cross-Exchange Perpetual Arbitrage

Hermes: when working in this repo, follow these rules.

## Project Rules

- **Language:** TypeScript strict, ESM only. No `any` without justification.
- **Test runner:** Vitest. Run `npm test` before declaring anything done.
- **Monorepo:** npm workspaces (`apps/*`, `packages/*`). Shared code lives in `@arb/shared`.
- **Architecture:** Adapter pattern for exchanges. Business logic never imports exchange SDKs directly.
- **Convention:** Config via env vars, secrets NEVER committed. See `.env.example`.
- **State machine:** Opportunities, positions, and exchange health all have explicit state machines (see ARCHITECTURE.md).
- **Execution:** IOC limit orders only. No naked market orders. Partial fills → rescue flow.
- **Safety:** Kill switches required before live. Dry-run first, always.

## Loop Workflow

When working autonomously:
1. Read `STATE.md` to know current progress
2. Pick the next undone item
3. Implement it
4. Run `npm test`
5. Update `STATE.md`
6. Commit with a descriptive message
7. Report what was done and what's next

## Key Thresholds (DO NOT CHANGE WITHOUT USER APPROVAL)

| Parameter | Value |
|-----------|-------|
| Minimum net return | 10 bps |
| Market data stale | 500 ms |
| Market data unhealthy | 2,000 ms |
| Rescue cost cap | 10 bps |
| Default notional | $100 |

## Verbs

- `npm test` — run all tests
- `npm run typecheck` — type-check without emitting
- `npm run dev:api` — start API server
- `npm run dev:web` — start dashboard
